# posrelayd-server

## Описание
Relay-сервер, маршрутизирующий запросы между пользовательским приложением для управления и клиентом на удалённой машине.

**Удалённый клиент:**
https://github.com/symp1ex/POSRelayd

**Пользовательское приложение:**
https://github.com/symp1ex/POSRelayv

## Сборка и запуск

### Требования

- Go 1.25+
- PostgreSQL 15+

### Docker
1. Команда для сборки docker-образа
```bash
docker build -t prserver .
```
2. Создать на хосте каталог для хранения конфига и логов приложение, например **`/data/prserver/data`**


3. Прописать настройки подключения к БД в **`config.json`**, заменить мастер-ключи и поместить конфиг в созданный ранее каталог


4. Запустить контейнер примонтировав созданный каталог. Пример команды:
```bash
sudo docker run -d --network host -v /data/prserver/data/:/app/data --restart=always -e TZ=Etc/GMT-3 --name posrelayd prserver
```
### Windows
1. Команда для сборки исполняемого файла
```bash
go build -o posrelayd-server.exe -ldflags="-s -w -H windowsgui" -trimpath .
```
2. Создать рядом с исполняемым файлом каталог **`data`** и поместить туда отредактированный **`config.json`**

## Конфигурация

### config.json

```json
{
  "service": {
    "port": 22233,
    "client_key": "b5679e9e-b5b5-4eaf-bb99-83dba95f9f53",
    "admin_key": "b5679e9e-b5b5-4eaf-bb99-83dba95f9f53",
    "temp_pass_renewal_period": 90
  },
  "database": {
    "host": "localhost",
    "port": 5432,
    "db_name": "posrelayd",
    "user": "user",
    "password": "password"
  },
  "logs": {
    "log_level": "debug",
    "store_days": 21
  }
}
```

Общие параметры:
- `port`: порт, на котором будет запущено приложение
- `client_key`: мастер-ключ для подключения удалённого клиента
- `admin_key`: мастер-ключ для подключения пользовательского приложения
- `temp_pass_renewal_period`: период обновления временного пароля на удалённом клиенте

Параметры подключения к базе данных PostgreSQL:
- `host`: адрес подключения к серверу PostgreSQL
- `port`: порт подключения к серверу PostgreSQL
- `db_name`: имя базы данных
- `user`: имя пользователя PostgreSQL
- `password`: пароль

Параметры логирования:
- `log_level`: уровень логирования
- `store_days`: срок хранения логов (дни)

## Безопасность

### TLS

Relay-сервер не выполняет прикладное шифрование данных, передаваемых через
WebSocket. Конфиденциальность и целостность передаваемых данных должны
обеспечиваться на транспортном уровне с помощью TLS.

TLS — это обязательное требование безопасности для развертывания в производственной среде, а не дополнительная опция.

### Blacklist

На данном этапе защита от анализа протокола или подбора мастер-ключей настроена крайне агрессивно, блокировка происходит по IP-адресу после нескольких неудачных попыток. 
Для удаления IP-адреса из чёрного списка, придётся вручную редактировать в БД таблицу **`blacklist`**
